github pages 활용하기

  1. 새로운 Repositories를 생성한다.

  2. Repositories의 이름은 URL을 생성하는 기준이다. 사용자ID.github.io를 추천한다. (참조 csbcp.github.io)

  3. github의 url을 복사하고 _config.yml의 # Deployment로 이동한다.

  4. type, repo, branch를 등록한다.

  5. github에 deploy 이전에 필요한 패키지를 설치한다.

  6. github에 파일을 올릴때에는 정적파일 형태로 deploy를 해야하므로 generate를 실행한다.

  7. deploy에 필요한 정보를 설정한다.

  8. github에 deploy 한다.

Hexo를 이용해서 github.io blog 만들기

Hexo를 이용해서 github.io blog 만들기

  • Hexo는 무엇인가요? (참조 https://hexo.io/ko/docs/)
    Hexo는 빠르고 간단하고 파워풀한 블로그 프레임워크입니다.
    Markdown(또는 다른 언어)을 사용하여 포스트를 작성하면 Hexo는 금세 멋진 테마를 가미해서 정적인 파일을 생성해 드립니다.

  1. 구름 사이트 접속 이후 계정을 생성한다. (참조 https://ide.goorm.io/?plan=ZnJlZQ== )

  2. 새로운 컨테이너를 생성클릭한다.

    1. 이름, 설명을 등록하고 소프트웨어 스택은 Node.js를 선택한다.
    2. 컨테이서 생성을 클릭한다.
    3. 새로운 컨테이너가 생성되면 새로운 구름IDE의 페이지가 실행된다.
    4. 구름IDE의 좌측은 폴더가 아랫쪽은 터미널이 실행된다.

  3. 구름IDE의 터미널에 Hexo를 설치하기 위해서 아래의 명령을 실행한다. -g global

  4. Hexo를 초기화한다. hexoblog는 구름IDE의 좌측에 생성되는 폴더이름이다.

  5. 현재 디렉토리 확인 이후 4번에서 생성한 폴더로 이동한다.

  6. 터미널에 npm install을 명령어를 실행한다.

  7. themes를 설치한다. (참고 https://github.com/ppoffice/hexo-theme-icarus)

  8. 구름IDE의 터미널에 .git를 삭제하고 git clone을 실행한다. 좌측에 생성된 hexo-theme-icarus를 확인한다. 편의상 폴더 이름이 길어서 hexoblog로 변경했다.

    theme: hexoblog

    root@goorm:/workspace/blog/hexoblog/themes# hexo server -p 3000
    ```
